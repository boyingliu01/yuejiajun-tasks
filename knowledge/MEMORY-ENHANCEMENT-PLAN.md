# 越甲军记忆增强方案

## 问题现状

**症状：**
- 龙虾成员频繁"失忆"，需要重复相同信息
- 任务完成后无记录，下次遇到同样问题重新摸索
- 团队成员之间知识不共享，重复劳动
- ArkClaw 曾因"快速模式"放鸽子（声称完成但实际未完成）

**根因：**
- 缺少持久化记忆机制
- 没有强制记忆写入流程
- 缺少团队共享知识库
- 会话上下文限制导致信息丢失

---

## 推荐方案：三层记忆架构

基于业界最佳实践（GraphRAG、MemMachine、Collaborative Memory 等）和 OpenClaw 原生能力，设计如下架构：

```
┌─────────────────────────────────────────────────────────┐
│  Layer 3: 全局层（团队共享记忆）                          │
│  ────────────────────────────────────────────────────   │
│  载体：GitHub 仓库 + 共享 MEMORY.md                       │
│  内容：团队决策、项目知识、最佳实践、教训记录              │
│  访问：所有成员可读写，永久保存                           │
│  更新：任务完成后强制写入                                 │
└─────────────────────────────────────────────────────────┘
                            ↓ 固化/提炼
┌─────────────────────────────────────────────────────────┐
│  Layer 2: 会话层（个体持久记忆）                          │
│  ────────────────────────────────────────────────────   │
│  载体：OpenClaw memory/*.md + sessions.jsonl            │
│  内容：每日日志、任务记录、学习心得、错误教训             │
│  访问：每个成员独立维护，跨会话保留                       │
│  更新：每次会话自动写入，每日整理                        │
└─────────────────────────────────────────────────────────┘
                            ↓ 加载/注入
┌─────────────────────────────────────────────────────────┐
│  Layer 1: 上下文层（临时工作记忆）                        │
│  ────────────────────────────────────────────────────   │
│  载体：当前对话窗口 + 临时文件                            │
│  内容：当前任务的中间状态、短期上下文                      │
│  访问：会话期间有效，结束后清理                          │
│  更新：实时写入                                          │
└─────────────────────────────────────────────────────────┘
```

## 实施方案

### 阶段一：基础建设（本周完成）

#### 1.1 GitHub 仓库结构

```
yuejiajun-tasks/
├── MEMORY.md                    # 团队共享记忆（索引 + 核心认知）
├── team/
│   ├── members.md               # 成员档案（能力、偏好、联系方式）
│   └── protocols.md             # 团队协作协议
├── tasks/
│   ├── YYYY-MM-DD-task-xxx.md   # 任务记录模板
│   └── task-log.md              # 任务总览
├── learnings/
│   ├── LEARNINGS.md             # 团队教训记录
│   ├── ERRORS.md                # 错误记录
│   └── BEST-PRACTICES.md        # 最佳实践
├── knowledge/
│   ├── model-configs.md         # 大模型配置知识
│   ├── dev-environment.md       # 开发环境配置
│   └── security-checklist.md    # 安全检查清单
└── archives/                    # 归档（旧任务、旧日志）
```

#### 1.2 记忆写入流程（强制）

**任务完成后必须执行：**
```markdown
1. 在 GitHub 任务目录创建记录文件
2. 更新 MEMORY.md 索引
3. 更新 learnings/ 相关文件（如有新教训）
4. 在群里通知"任务完成 + GitHub 链接"
```

**模板：**
```markdown
# 任务记录：[任务名称]

- 执行者：@xxx
- 完成时间：YYYY-MM-DD HH:mm
- 耗时：X 小时
- GitHub 提交：[链接]

## 执行过程
（关键步骤、遇到的问题、解决方案）

## 验证证据
（截图、日志、输出文件链接）

## 经验教训
（值得记录的经验、教训、最佳实践）

## 待改进
（下次可以做得更好的地方）
```

#### 1.3 启动检查清单

**每次开始工作前必读：**
```markdown
1. 读 MEMORY.md（团队共享记忆）
2. 读 memory/YYYY-MM-DD.md（今日日志）
3. 读 memory/YYYY-MM-DD-1.md（昨日日志）
4. 检查 GitHub 任务看板
```

### 阶段二：自动化增强（下周开始）

#### 2.1 记忆固化机制

**每日下班前自动执行：**
```bash
# 伪代码
1. 收集当日所有 memory/*.md 文件
2. 提炼关键信息到 MEMORY.md
3. 归档旧日志到 archives/
4. 生成日报发送到群里
```

#### 2.2 向量检索（可选）

如果团队记忆文件超过 100 个，建议引入向量检索：
- 使用 ChromaDB / Milvus / FAISS
- 将记忆文件向量化存储
- 查询时自动检索相关记忆

#### 2.3 记忆检查工具

开发一个脚本，检查记忆完整性：
```bash
# 检查项
- MEMORY.md 是否更新
- 今日日志是否存在
- 任务记录是否完整
- 教训是否记录
```

---

### 阶段三：团队协作增强（长期）

#### 3.1 共享记忆协议

参考 **Collaborative Memory** 框架：
- **私有记忆**：每个成员的独立日志
- **共享记忆**：团队决策、项目知识
- **访问控制**：敏感信息权限管理

#### 3.2 记忆同步机制

- 定期（每周）同步各成员记忆
- 发现重复问题时自动提醒"已有解决方案"
- 跨成员知识传递（A 的经验 B 可以直接用）

## 成功标准

✅ **不再失忆：**
- 每个成员启动时能读到昨日上下文
- 重复问题能找到历史解决方案
- 任务完成有记录可查

✅ **知识共享：**
- 新成员加入能快速了解团队知识
- 一人踩坑，全员避坑
- 最佳实践可复用

✅ **可追溯：**
- 任何决策有记录
- 任何错误有教训
- 任何任务有证据
