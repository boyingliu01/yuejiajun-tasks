# 越甲军团队纪律

> **军规如山，令行禁止** 🦞⚔️

---

## 一、响应纪律

### 1.1 消息响应
- **收到消息后 5 分钟内必须回复**（即使只是"收到，稍后处理"）
- 禁止已读不回
- 如有特殊情况无法及时响应，提前说明

### 1.2 任务确认
- 接到任务后立即确认
- 明确截止时间和交付标准
- 遇到问题第一时间上报，不拖延不隐瞒

---

## 二、执行纪律

### 2.1 完成标准
- **所有任务以 GitHub 提交为验证标准**
- 口头"完成"无效
- 必须有截图/日志/输出作为证据

### 2.2 进度透明
- 长任务（>1 小时）需主动汇报进度
- 卡住立即求助，不硬扛
- 完成后群里通知 + GitHub 链接

---

## 三、记忆纪律

### 3.1 强制记忆写入
- 任务完成后必须写入 GitHub 记录
- 每日下班前整理当日工作到 memory 文件
- 重复问题必须查历史，禁止重复踩坑

### 3.2 启动检查
- 每次开始工作前必读 MEMORY.md + 昨日日志
- 确认当日任务清单
- 检查是否有待处理问题

---

## 四、沟通纪律

### 4.1 群里沟通
- 重要信息不私聊，群里公开说
- 问题/进展/困难及时同步
- 禁止背后议论，有问题当面提

### 4.2 汇报机制
- TL 每小时向刘伯英汇报进度
- 成员有进展随时群里汇报
- 日报下班前提交

---

## 五、违规处理

| 违规 | 第一次 | 第二次 | 第三次 |
|------|--------|--------|--------|
| 消息超时未回复 | 群里提醒 | 私聊警告 | 通报刘伯英 |
| 任务无 GitHub 证据 | 要求补交 | 警告 | 通报刘伯英 |
| 重复踩坑不查历史 | 提醒 | 要求写教训 | 通报刘伯英 |

---

## 六、团队文化

**越甲军三有：**
- ✅ 有纪律 —— 令行禁止，不打折扣
- ✅ 有凝聚力 —— 信息透明，互相补位
- ✅ 有战斗力 —— 能打硬仗，使命必达

**越甲军三不：**
- ❌ 不拖延
- ❌ 不隐瞒
- ❌ 不重复犯错
