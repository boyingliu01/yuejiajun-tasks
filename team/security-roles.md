# 越甲军安全角色分工

> 红蓝对抗机制：以攻促防，提升整体安全水平

---

## 角色分工

### 🔴 红队（攻击方）

**负责人：** 开发 Claw

**职责：**
- 对 ArkClaw 服务器进行安全测试
- 模拟真实攻击场景
- 发现安全漏洞并报告
- 提供修复建议

**技能要求：**
- 渗透测试工具（nmap、Metasploit、Burp Suite 等）
- 漏洞扫描（OpenVAS、Nessus 等）
- Web 安全测试（SQL 注入、XSS、CSRF 等）
- 社会工程学测试

**工具清单：**
```bash
# 信息收集
nmap - 端口扫描
whois - 域名信息查询
theHarvester - 邮箱/子域名收集

# 漏洞扫描
OpenVAS - 漏洞扫描
Nikto - Web 服务器扫描
SQLmap - SQL 注入测试

# Web 安全
Burp Suite - Web 代理/测试
OWASP ZAP - Web 漏洞扫描

# 密码测试
hydra - 暴力破解
john - 密码破解

# 后渗透
Metasploit - 漏洞利用框架
```

## 红蓝对抗流程

### 阶段 1：基线检查（蓝队）

```
ArkClaw 完成全面安全检查
输出安全基线报告
修复已知高危漏洞
```

### 阶段 2：攻击测试（红队）

```
开发 Claw 制定测试计划
执行渗透测试
记录攻击路径和发现的漏洞
输出红队测试报告
```

### 阶段 3：修复加固（蓝队）

```
ArkClaw 根据红队报告修复漏洞
加固安全措施
输出修复报告
```

### 阶段 4：复测验证（红队）

```
开发 Claw 复测已修复漏洞
确认修复有效
输出验证报告
```

### 阶段 5：常态化（持续）

```
每月一次红蓝对抗
定期安全培训
更新安全基线
```

## 测试规则（RoE）

### 允许的操作

- ✅ 端口扫描和服务识别
- ✅ 漏洞扫描
- ✅ Web 应用测试
- ✅ 暴力破解测试（限测试账户）
- ✅ 社会工程学测试（需提前告知）

### 禁止的操作

- ❌ 删除或修改生产数据
- ❌ 拒绝服务攻击（DoS/DDoS）
- ❌ 物理安全测试
- ❌ 针对第三方的攻击
- ❌ 未经授权的横向移动

### 时间窗口

- **首次测试：** 2026-04-17 ~ 2026-04-24
- **测试时间：** 工作日 10:00-18:00（避免影响正常业务）
- **紧急联系：** 群里实时沟通
